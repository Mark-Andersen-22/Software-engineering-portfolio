## CS250 Software Development Lifecycle
This folder contains assignments and portfolio artifacts from my CS250 Software Development Lifecycle course.

## Course Reflection

### How do I interpret user needs and implement them into a program?
Throughout this course, I learned tha understanding the user's needs is the foundation of building successful software. Creating user stories helps break down those needs into clear, manageable requirements that guide development. By focusing on what the user wants to accomplish, I can design features that solve real problems while staying organized throughout the development process.

### How do I approach developing programs?
I now approach software development as an interative process rather than trying to build everything at once. Agile practices such as sprint planning, daily communication, sprint reviews, and retrospectives help keep projects organized and allow for continuous improvement. In future projects, I plan to continue using these Agile practices because they encourage collaboration, adaptability, and regular feedback.

### What does it mean to be a good team member in software development?
Being a good team member means communicating clearly, respecting different ideas, completing assigned work on time, and helping others when needed. Software development is a collaborative effort, and successful teams depend on trust, accountability, and a willingness to learn from one another. I believe these qualities are just as important as technical skills.
